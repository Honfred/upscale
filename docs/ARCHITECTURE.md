# Архитектура AnimeUpscale

## Стек

- **Tauri 2** (Rust) — нативная оболочка, IPC-команды, файловая система, запуск sidecar-процессов.
- **Svelte 5** + Vite — UI (рантайм-состояние через `$state`/runes, см. `src/lib/stores.svelte.ts`).
- Никакого бэкенд-сервера: вся обработка видео идёт локально через внешние CLI-инструменты,
  запускаемые Rust-кодом как sidecar-процессы (`tauri-plugin-shell`).

## Пайплайн обработки

Апскейл видео — тяжёлая операция, поэтому видео режется на **сегменты** (чанки) и
обрабатывается пофазово, чтобы:
- показывать прогресс и ETA по каждому сегменту, а не только "в целом";
- уметь возобновлять/перезапускать обработку с места сбоя, не теряя уже готовые куски;
- ограничивать пиковое использование диска (временные файлы одного сегмента, а не всего фильма).

Условно шаги пайплайна (см. `src-tauri/src/pipeline/`):

1. **probe** — `ffprobe` определяет параметры исходника (разрешение, fps, длительность, кодек).
2. **segment** — исходное видео режется на сегменты фиксированной длины (`ffmpeg -c copy`, без
   перекодирования).
3. **decode** — кадры сегмента извлекаются в PNG/иной промежуточный формат.
4. **upscale** — `realesrgan-ncnn-vulkan` (модель `realesr-animevideov3-x{2,3,4}`) апскейлит кадры.
5. **interpolate** — `rife-ncnn-vulkan` (модель `rife-v4.6`) добавляет промежуточные кадры для
   повышения fps.
6. **encode** — `ffmpeg` с NVENC кодирует обработанные кадры сегмента обратно в видео.
7. **concat** — готовые сегменты склеиваются в итоговый файл (`ffmpeg concat`, без перекодирования).
8. **cleanup** — временные файлы сегмента удаляются.

Прогресс каждой фазы транслируется в UI через Tauri-события (`src-tauri/src/pipeline/progress.rs`).

## Одна GPU-джоба

Real-ESRGAN и RIFE (ncnn-vulkan) и NVENC-энкодинг используют один и тот же GPU. Пайплайн
намеренно выполняет **не более одной** GPU-интенсивной операции одновременно (upscale ИЛИ
interpolate ИЛИ encode), чтобы не конкурировать за видеопамять и не деградировать по скорости
из-за контекст-свитчинга Vulkan/NVENC на одной карте. Параллелизм есть только между
CPU-фазами (decode/segment/concat/cleanup) и следующей GPU-фазой соседнего сегмента.

## Sidecar-бинарники

Приложение не линкует ffmpeg/Real-ESRGAN/RIFE статически в Rust — они запускаются как внешние
процессы (Tauri `externalBin`, см. `src-tauri/tauri.conf.json`):

| Имя (externalBin)         | Назначение                                  |
|----------------------------|----------------------------------------------|
| `realesrgan-ncnn-vulkan`   | апскейл кадров (Vulkan)                      |
| `rife-ncnn-vulkan`         | интерполяция кадров (Vulkan)                 |
| `ffmpeg`                   | декод/энкод (NVENC), резка/склейка сегментов |
| `ffprobe`                  | анализ исходного файла                       |

Бинарники и модели не хранятся в git (см. `.gitignore`) — они скачиваются под пиновые версии
скриптами `scripts/fetch-sidecars.{sh,ps1}` и `scripts/fetch-models.sh` (см. корневой README) и
кладутся в `src-tauri/binaries/` и `src-tauri/resources/models-*/` перед сборкой.

## Известные ограничения

- Sidecar-бинарники требуют Vulkan-совместимый GPU (Real-ESRGAN/RIFE) и NVIDIA GPU с NVENC для
  аппаратного энкода.
- `realesrgan-ncnn-vulkan.exe` (Windows-сборка) динамически линкуется с `vcomp140.dll`
  (OpenMP-рантайм MSVC). На машинах без установленного Visual C++ Redistributable запуск может
  падать с ошибкой отсутствия DLL — см. предупреждение в `scripts/fetch-sidecars.ps1`.
